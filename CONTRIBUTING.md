# Contributing to MoodWav

First off, thank you for considering contributing to MoodWav! It's people like you that make the open source community such a fantastic place to learn, inspire, and create. Any contributions you make are greatly appreciated.

## How Can I Contribute?

There are many ways you can contribute to MoodWav:

- Reporting Bugs
- Suggesting Enhancements
- Pull Requests

## Getting Started

### Prerequisites

- Familiarity with Next.js, TailWindCSS, Supabase, and the Spotify Web API
- Node.js and npm installed

### Setting Up Your Environment

1. Fork the repository - This creates your own copy of the project.
2. Clone your fork - Downloads a copy to your local machine.
3. Install dependencies - Run `npm install` in the project root.
4. Create a branch - Use a clear, descriptive name for your branch.

## Making Changes

- Follow the coding standards - Ensure your code adheres to AirBnB's style guide.
- Write meaningful commit messages - Describe what you did and why.
- Test your changes - Make sure your changes do not break existing functionality.
- Write documentation - Update the documentation to reflect your changes, if necessary.

## How to Submit a Pull Request

1. Push your changes to your fork.
2. Open a pull request - Provide a clear description of the problem and solution. Include the relevant issue number if applicable.
3. Wait for feedback - The project maintainers will review your PR and possibly request changes or provide feedback.
4. Make any requested changes - This can be an iterative process.
5. Merge! - Once your PR is approved, a maintainer will merge it into the main codebase.

## Reporting Bugs

When you are creating a bug report, include as many details as possible. Fill out the required template, the information it asks for helps us resolve issues faster.

## Suggesting Enhancements

Enhancement suggestions are always welcome. Don't hesitate to suggest an idea, but keep in mind that it might not align with the overall project goals.
